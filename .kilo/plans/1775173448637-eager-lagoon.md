# LiquiTask AI Enhancement & Task Management Plan

## Overview

This plan outlines comprehensive AI enhancements and task management improvements for LiquiTask v1.3.6, focusing on AI-powered task cleanup, duplicate detection, intelligent organization, and expanded AI capabilities.

---

## Phase 1: AI Task Management & Cleanup (Priority: HIGH)

### 1.1 Duplicate Task Detection & Merging

**Files to modify:**

- `src/services/aiService.ts` - Add `detectDuplicates()` method
- `src/services/taskCleanupService.ts` - **NEW FILE** - Task cleanup service
- `components/settings/AiSettings.tsx` - Add cleanup settings UI
- `components/TaskFormModal.tsx` - Add duplicate warning

**Implementation:**

- Add AI-powered duplicate detection that analyzes:
  - Title similarity (semantic matching via AI)
  - Summary/content overlap
  - Tags and metadata similarity
  - Project context
- Create `TaskCleanupService` with methods:
  - `detectDuplicates(allTasks): Promise<DuplicateGroup[]>` - Returns groups of potential duplicates with confidence scores
  - `suggestMerge(duplicateGroup): Promise<MergeSuggestion>` - AI suggests which task to keep and how to merge
  - `executeMerge(suggestion): void` - Performs the merge (keeps one, archives others, migrates subtasks/links)
- Add settings toggle for "Auto-detect duplicates on task creation"
- Show duplicate warnings in TaskFormModal when creating/editing

**New Types:**

```typescript
interface DuplicateGroup {
  id: string;
  tasks: Task[];
  confidence: number; // 0-1
  reasons: string[]; // AI-explained reasons
}

interface MergeSuggestion {
  keepTaskId: string;
  archiveTaskIds: string[];
  mergedFields: Partial<Task>;
  reasoning: string;
}
```

## Key Technical Considerations

### AI Context Enhancement

- Current AI context is limited to projects and priorities
- Expand to include: task history, completion rates, user patterns
- Cache AI context for faster response times

### Error Handling & Fallbacks

- All AI operations should have graceful fallbacks
- User should always be able to override AI suggestions
- Clear explanations for AI decisions

### Privacy & Local Processing

- Ollama support enables full offline AI
- No task data leaves device when using local AI
- Clear indicators for cloud vs local processing

### Undo & Reversibility

- All AI-driven changes should be undoable
- Preview changes before applying
- Batch undo for AI operations

## Key Design Principles

1. **Progressive Enhancement** - Core functionality works without JS/animations, enhanced with motion
2. **Mobile-First** - Design for mobile, enhance for desktop
3. **Accessibility-First** - WCAG AA compliance as baseline, AAA where practical
4. **Performance Budget** - Keep bundle under 400KB, TTI under 3s on 3G
5. **Consistent Motion** - Use same easing curves and durations across all animations
6. **Meaningful Feedback** - Every user action has visible, timely feedback

## Testing Strategy for UI/UX

1. **Visual Regression:** Screenshot comparison for key components
2. **Accessibility Audit:** axe-core automated tests + manual screen reader testing
3. **Responsive Testing:** Test on 320px, 768px, 1024px, 1440px, 2560px viewports
4. **Performance Testing:** Lighthouse scores (target: 90+ Performance, 100 Accessibility)
5. **Motion Testing:** Verify `prefers-reduced-motion` disables all non-essential animations
6. **Keyboard Testing:** Full tab navigation through all interactive elements

## Key Design Principles

1. **Progressive Enhancement** - Core functionality works without JS/animations, enhanced with motion
2. **Mobile-First** - Design for mobile, enhance for desktop
3. **Accessibility-First** - WCAG AA compliance as baseline, AAA where practical
4. **Performance Budget** - Keep bundle under 400KB, TTI under 3s on 3G
5. **Consistent Motion** - Use same easing curves and durations across all animations
6. **Meaningful Feedback** - Every user action has visible, timely feedback

---

## Testing Strategy for UI/UX

1. **Visual Regression:** Screenshot comparison for key components
2. **Accessibility Audit:** axe-core automated tests + manual screen reader testing
3. **Responsive Testing:** Test on 320px, 768px, 1024px, 1440px, 2560px viewports
4. **Performance Testing:** Lighthouse scores (target: 90+ Performance, 100 Accessibility)
5. **Motion Testing:** Verify `prefers-reduced-motion` disables all non-essential animations
6. **Keyboard Testing:** Full tab navigation through all interactive elements
