# Fix NotificationService Tests & Disable macOS/Linux Releases

## 1. Fix notificationService.test.ts - MockNotification spy issue

### Problem

The `MockNotification` class is not a vitest spy, so `expect(MockNotification).toHaveBeenCalledWith(...)` fails with "is not a spy or a call to a spy!".

### Solution

Wrap `MockNotification` with `vi.fn()` to make it spyable while keeping it constructable:

```typescript
class MockNotificationClass {
  static lastInstance: MockNotificationClass | null = null;
  onclick: (() => void) | null = null;
  static requestPermission = vi.fn(() => Promise.resolve('granted'));
  static permission = 'granted';
  constructor(_title: string, _options?: NotificationOptions) {
    MockNotificationClass.lastInstance = this;
  }
}

// Spy-wrapped constructor for `expect().toHaveBeenCalledWith()` assertions
const MockNotification = vi.fn(function (title: string, options?: NotificationOptions) {
  return new MockNotificationClass(title, options);
}) as any;
MockNotification.requestPermission = MockNotificationClass.requestPermission;
MockNotification.permission = MockNotificationClass.permission;
Object.defineProperty(MockNotification, 'lastInstance', {
  get: () => MockNotificationClass.lastInstance,
});
```

### Files to modify

- `src/services/__tests__/notificationService.test.ts` — Replace the `MockNotification` class with the spy-wrapped version above
