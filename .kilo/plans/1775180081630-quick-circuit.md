# AI Task Reorganization & Smart Optimization Features

## Overview

Add AI-powered features to reorganize, merge, group, and optimize the entire task collection. The existing AI infrastructure (`aiService.ts`, `taskCleanupService.ts`) already has `clusterTasks`, `detectDuplicates`, `suggestMerge`, `categorizeTasks`, and `suggestPriorities` — but none are wired into the UI as actionable features. This plan activates them.

---

## 1. New AI Features

### 1.1 "Smart Reorganize" — AI Task Clustering & Grouping

**What:** AI analyzes all tasks, groups them into themed clusters (e.g., "Backend API work", "UI redesign", "Bug fixes"), and suggests project/workspace reorganization.

**How:**

- Call `aiService.clusterTasks(allTasks, context)` to get `TaskCluster[]` (each cluster has a theme/title and task IDs)
- Present clusters in a modal with preview
- User can: Accept all (creates new projects/workspaces per cluster, moves tasks), Accept selected, or Dismiss
- Each cluster becomes a new project or the tasks get retagged with the cluster theme

**Files:**

- New: `src/components/AIReorganizeModal.tsx` — Modal showing clusters, accept/dismiss per cluster
- Modify: `App.tsx` — Add handler to open modal, wire to AI menu
- Modify: `aiService.ts` — Ensure `clusterTasks` prompt is robust for large task sets

### 1.2 "Convert to Subtasks" — Smart Task Merging

**What:** AI identifies tasks that are actually subtasks of other tasks (e.g., "Fix login button" is a subtask of "Redesign auth flow") and suggests converting them.

**How:**

- Use existing `taskCleanupService.analyzeRedundancy()` which already detects `subset` type (task is subtask of another)
- Add new AI-powered analysis: for each task pair, ask AI if one should be a subtask of the other
- Present suggestions in a modal: "Move 'Fix login button' as subtask of 'Redesign auth flow'?"
- On accept: remove standalone task, add as `Subtask` to parent, preserve metadata (assignee, tags) in parent's summary

**Files:**

- New: `src/components/AISubtaskSuggestionsModal.tsx`
- Modify: `src/services/taskCleanupService.ts` — Add `suggestSubtaskConversions(allTasks, context)` method
- Modify: `App.tsx` — Add handler

### 1.3 "Auto-Group by AI" — Smart Project Assignment

**What:** AI reassigns tasks to the most appropriate project/workspace based on content, tags, and context.

**How:**

- For each task, AI evaluates which project it best belongs to
- Suggests moves: "Move 'Update API docs' from 'General' to 'Backend API'"
- Batch apply with confirmation

**Files:**

- Modify: `aiService.ts` — Add `suggestProjectReassignment(allTasks, projects, context)` method
- New: `src/components/AIProjectAssignmentModal.tsx`
- Modify: `App.tsx`

### 1.4 "Smart Merge Duplicates" — Enhanced Merge UI

**What:** Replace the current heuristic-only "Cleanup Tasks Now" button with a full AI-powered merge flow.

**How:**

- Call `taskCleanupService.detectDuplicates()` → get duplicate groups
- For each group, call `taskCleanupService.suggestMerge()` → get which to keep, what to merge
- Show merge preview: "Merge 3 duplicate tasks into 'Update homepage design'" with merged fields highlighted
- One-click "Apply All Merges" or review individually

**Files:**

- New: `src/components/AIMergeDuplicatesModal.tsx`
- Modify: `App.tsx` — Replace current `handleCleanupTasks` with open merge modal
- Modify: `AiSettings.tsx` — "Cleanup Tasks Now" button opens merge modal instead of heuristic cleanup

### 1.5 "AI Insights Dashboard" — Smart Task Health Overview

**What:** A dedicated panel showing AI-generated task health metrics:

- Task distribution by project (are some overloaded?)
- Stale tasks (no updates in X days)
- Overdue risk predictions
- Suggested task priorities based on deadlines and dependencies
- Bottleneck detection (tasks blocking others)

**How:**

- Extend existing `aiService.generateInsights()` with richer analysis
- New dedicated view accessible from command palette and AI menu

**Files:**

- Modify: `aiService.ts` — Enhance `generateInsights` prompt
- New: `src/components/AIHealthDashboard.tsx`
- Modify: `App.tsx` — Add route/view

## 5. Key Technical Considerations

- **Large task sets:** AI calls can be slow. Use streaming/progress indicators. Chunk tasks for LLM context limits.
- **Undo support:** All AI operations should be reversible. Use existing undo stack pattern.
- **Confirmation:** Never auto-apply AI changes without user review. Always show preview.
- **Provider agnostic:** Works with both Gemini and Ollama.
- **Error resilience:** If AI fails, fall back to heuristics. Show graceful error.
- **Cost awareness:** For Gemini API users, batch operations should show estimated token/cost info.
